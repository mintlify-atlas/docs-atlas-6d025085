# Welcome to Scope Rule Architects

A collection of specialized Claude Code agents that enforce the **Scope Rule** architectural pattern across different frontend frameworks. These agents help developers make consistent architectural decisions and maintain clean, scalable codebases.

## What is the Scope Rule?

The Scope Rule is a fundamental architectural principle that determines code placement based on usage scope:

**"Scope determines structure"**

<CardGroup cols={2}>
  <Card title="2+ Features" icon="share-nodes" color="#94cb04">
    Code used by multiple features **MUST** go in global/shared directories
  </Card>
  <Card title="1 Feature" icon="folder" color="#545454">
    Code used by a single feature **MUST** stay local in that feature
  </Card>
</CardGroup>

<Note>
  This rule is **absolute and non-negotiable** — no exceptions.
</Note>

## Quick Example

<Tabs>
  <Tab title="Question">
    ```
    I have a ProductCard component used in shop, cart, and wishlist pages.
    Where should I place it?
    ```
  </Tab>
  <Tab title="Answer">
    ```
    Used by 3 features (shop, cart, wishlist) → 2+ features
    
    ✅ MUST place in: shared/components/ProductCard
    
    Reasoning: The Scope Rule dictates that any code used by 2 or more 
    features must be placed in global/shared directories. This ensures 
    perfect traceability and prevents duplication.
    ```
  </Tab>
</Tabs>
